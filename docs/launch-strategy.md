# Wrangle Launch Strategy — Anti-Marketing Playbook

## Context

Wrangle is a native macOS markdown editor purpose-built for developers working with AI agents (Claude Code, skills, system prompts, MCP configs). It's launching on the Mac App Store. The goal is a low-effort, authentic, anti-marketing approach inspired by Passive Rebel's "skip the bullshit" ethos — personal, raw, polarizing, and magnetic.

The core insight: **Wrangle doesn't need to appeal to everyone. It needs to be undeniable to the ~10,000 people who live in CLAUDE.md files and system prompts all day.**

---

## The Anti-Marketing Positioning

### One-liner
**"The markdown editor that actually knows what a CLAUDE.md file is."**

### The Polarizing Angle
Every other markdown editor treats your AI config files like dumb text. Wrangle knows the difference between a `SKILL.md` and a shopping list. If you don't know what that means, this app isn't for you. That's the point.

### Voice & Tone
- First person. You built this because you needed it.
- Casual, developer-to-developer. No "we're excited to announce."
- Opinionated. "VS Code is not a markdown editor. It's a code editor with a markdown preview bolted on."
- Short. Punchy. Screenshot-heavy. Let the app speak.

---

## Launch Channels (Low Effort, High Signal)

### 1. Build-in-Public Twitter/X Thread (Day 0)
Write a single thread. Not a launch announcement — a story.

**Format:**
> "I've been writing CLAUDE.md files for months. Every editor treats them like plain markdown. None of them understand XML-in-markdown, token counting, or that a `<tools>` block isn't just angle brackets.
>
> So I built one that does. It's called Wrangle. Here's what it looks like. [screenshots]
>
> It's $X on the Mac App Store. If you work with AI agents daily, you'll get it immediately. If you don't, that's cool — this wasn't built for you."

That last line is Passive Rebel energy. It's a filter. It makes the people who DO get it feel seen.

**Effort: 1 hour.** Write it, attach 4-6 screenshots, post it.

### 2. Targeted Reddit Posts (Week 1)
Not "check out my app" spam. Genuine problem→solution posts.

**Subreddits:** r/ClaudeAI, r/LocalLLaMA, r/MacApps, r/Anthropic

**Format:** Don't lead with the app. Lead with the pain.
> "Anyone else frustrated editing CLAUDE.md files in VS Code? The XML highlighting breaks, there's no token counting, and you can't see what your system prompt actually looks like rendered. I ended up building a native macOS editor specifically for this. [link if anyone wants it]"

Low-key. Helpful. Not salesy. People will ask questions. Answer them like a human, not a marketer.

**Effort: 30 min per post.** Do 3-4 posts across a week.

### 3. Claude Code Community / Anthropic Discord
You are literally the target user building for target users. Show up where they hang out. Share screenshots. Answer questions about AI workflows. Mention you built an editor when it's naturally relevant.

**Effort: Passive.** Just be present.

### 4. One Blog Post / Personal Site (Week 1-2)
Title ideas (pick the most "you"):
- "I Built a Markdown Editor Because VS Code Kept Breaking My System Prompts"
- "Why AI Config Files Deserve a Real Editor"
- "The App I Wish Existed 6 Months Ago"

Keep it short (500-800 words). The structure:
1. The problem (editing AI files sucks in generic editors)
2. What I tried (Typora, Obsidian, VS Code — what fell short)
3. What I built (Wrangle, with screenshots)
4. Where to get it (Mac App Store link)

This becomes your canonical link you share everywhere.

**Effort: 2-3 hours once.**

### 5. Hacker News "Show HN" (Week 2)
**Title:** "Show HN: Wrangle – A macOS markdown editor built for AI agent config files"

HN loves: native apps, opinionated tools, solo dev projects, tools built out of genuine need. Wrangle checks every box.

Keep the description factual and technical. Don't oversell. The HN crowd will tear apart anything that feels like marketing, but they'll champion a dev tool built out of real frustration.

**Effort: 30 min to write.** Then engage with comments for a day.

## The "Anti-Ad" Campaign (Optional, ~$50-200)

If you want to run something paid with minimal effort:

### Twitter/X Promoted Post
Take your best-performing organic tweet and promote it. Don't create ad copy. Just boost the thing that already resonated.

**Target:** People who follow @AnthropicAI, @ClaudeAI, AI developer accounts.

**Budget:** $50-100 for a week. See what happens.

### One Visual Ad (If You Want to Be Cheeky)
A single image ad. Black background, monospace font:

```
your CLAUDE.md file
deserves better than
VS Code's markdown preview.

wrangle — mac app store
```

That's it. No gradients, no app screenshots, no "AI-powered" buzzwords. Anti-design. Passive Rebel energy. It'll stand out in a feed full of polished SaaS ads precisely because it looks like a terminal output.

---

## What NOT to Do

- **No Product Hunt launch** (unless you genuinely want to). It's a time sink for a niche macOS app. The audience isn't there.
- **No influencer outreach.** Your users are too technical to be influenced by a YouTuber holding your app.
- **No feature comparison tables.** Don't compete on features. Compete on understanding.
- **No "we" language.** You're one person who built a thing. Own that.
- **No launch day pressure.** This isn't a Kickstarter. The Mac App Store is always open. Launch quietly, let it build.

---

## The Long Game (Post-Launch, Zero Extra Effort)

- **Let the app be the marketing.** If Wrangle genuinely solves a problem, developers tell other developers. That's it.
- **Ship updates publicly.** Quick tweets: "Just shipped XML tag collapsing in Wrangle. If you edit system prompts with nested `<tools>` blocks, this one's for you." Screenshots attached. Natural, not performative.
- **Keep building in your niche.** The AI agent ecosystem is exploding. Every new Claude feature = a new reason someone needs Wrangle. You don't need to chase trends. The trends are coming to you.

---

## Summary: The Minimum Viable Launch

| Action | Effort | When |
|---|---|---|
| Twitter/X launch thread | 1 hour | Day 0 |
| 3-4 Reddit posts | 2 hours total | Week 1 |
| Blog post | 2-3 hours | Week 1-2 |
| Show HN | 30 min + engagement | Week 2 |
| Community presence | Ongoing, passive | Always |
| Optional: boost top tweet | $50-100 | Week 2+ |

**Total active effort: ~6-8 hours spread over 2 weeks.** Then just keep shipping and talking about what you ship.

---

## The Core Philosophy

Passive Rebel's insight applies perfectly here: **you don't need a brand strategy. You need to be so clearly, unapologetically for your specific person that they feel like you read their mind.**

Your specific person: a developer who has 15 CLAUDE.md files open, can't get token counts without a separate tool, and is tired of VS Code mangling their XML tags. Speak to that person like you're texting a coworker. That's the whole strategy.

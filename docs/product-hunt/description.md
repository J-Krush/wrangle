# Product Hunt Description

PH description limit: **260 characters** (including spaces).

Shown on the listing page under the tagline. Should do three jobs: (1) tell people what the app actually is, (2) hint at who it's for, (3) give a reason to click through.

Counted below each option.

---

## Option 1 — Three-in-one framing (Recommended)

> A native macOS workspace for developers running Claude Code, Gemini, and other AI agents. Markdown editor + embedded terminals + embedded browser with DevTools, in one window. Knows CLAUDE.md, SKILL.md, AGENTS.md. Apple Silicon, not Electron.

Chars: 257/260

Pros: covers the three-way combo explicitly. Name-drops the AI tools the audience uses. Ends on the "not Electron" filter — always lands with native-first devs.
Cons: slightly list-y. No emotional hook.

---

## Option 2 — Pain-first framing

> Tired of VS Code mangling your CLAUDE.md files? Wrangle is a native macOS editor that treats XML-in-markdown, token counts, and agent config files as first-class. Plus embedded terminals and a WebKit browser for the stuff your agent just shipped.

Chars: 254/260

Pros: opens on a relatable pain point. Matches the anti-marketing voice from `launch-strategy.md`.
Cons: takes longer to get to what the app is. PH scrollers are impatient.

---

## Option 3 — Positioning-first framing

> The markdown editor built for AI-native development. Edit CLAUDE.md files, run Claude / Gemini agents in tabbed terminals, inspect your work in an embedded browser. Built by one developer who lives in this workflow. macOS, Apple Silicon, $19 one-time.

Chars: 252/260

Pros: leads with positioning. Mentions price up front (filter + honesty). Solo-dev framing resonates on PH.
Cons: "AI-native" is slightly buzzword-y for the anti-marketing voice.

---

## Recommendation

**Go with Option 1.** It's the clearest statement of what you get. Save Option 2's pain-first framing for the first-comment maker post — that's where longer-form voice belongs.

# Phase 11 ‚Äî UI Design Contract

> Visual and interaction contract for Phase 11 (hide-when-empty + nested Bookmarks under Browsers). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Platform:** native macOS (AppKit/SwiftUI). No web, no Tailwind, no shadcn. Design tokens are SF Symbols, SwiftUI `.font(.system(size:weight:))`, and `Theme.*` (NSColor) ‚Äî not hex / CSS variables. The template shape is preserved; values are substituted for the native stack.

## Spacing Scale

Phase 11 introduces no new spacing tokens. It reuses the scale already locked across the sidebar and Project Overview. All values below are multiples of 2; multiples of 4 are the default step, with 2 reserved for tight sidebar list-row insets where density matters (per `.sidebarRowSize(.small)`).

| Token | Value | Usage |
|-------|-------|-------|
| xxs | 2px | Vertical padding inside `listRowInsets` tight rows (existing) |
| xs | 4px | Icon-to-label gaps; horizontal `listRowInsets`; chevron-to-title spacing |
| sm | 8px | Compact element spacing; bottom-bar horizontal padding |
| md | 12px | Overview card internal padding (`CardStyle` ‚Äî existing) |
| lg | 16px | Default element spacing; inline empty-state row padding (existing, being removed) |
| lgx | 24px | `ProjectOverviewView` top-level `VStack` spacing between sections (existing `spacing: 24`) |
| xl | 32px | Overview scroll-view padding (existing `.padding(32)`) |

Exceptions:
- `.sidebarRowSize(.small)` + `.defaultMinListRowHeight(20)` are already applied at `SidebarView.swift:100-101`. The nested Bookmarks sub-section inherits these ‚Äî no per-row height override.
- The **nested Bookmarks sub-header** (inside Browsers) uses `HStack(spacing: 4)` ‚Äî matches the existing `SidebarSectionHeader` chevron-to-title gap exactly. No indent added to signal nesting; chevron + container is the hierarchy cue (D-17 discretion + code_context recommendation).
- The **overview empty hero** uses `VStack(spacing: 12)` internally with `48px` vertical padding above/below the icon cluster (Claude's discretion ‚Äî matches the dense-utility voice and avoids the "huge landing-page empty state" aesthetic that would clash with the rest of the overview).

## Color

Phase 11 adds zero new colors. It consumes the existing palette. Listed below for the downstream consumers (planner, executor, checker, auditor) to verify no color additions slipped into implementation.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `Color(nsColor: Theme.chromeBackground)` ‚Äî dark: `#1c1c1c`, light: `windowBackgroundColor` | Window chrome; overview background |
| Secondary (30%) | `Color(nsColor: Theme.sidebarBackground)` ‚Äî dark: `#262626`, light: `#ececec` | Sidebar card; overview card fills (at `.opacity(0.8)` per existing `CardStyle`) |
| Accent (10%) | `Color.accentColor` (system) | Active filter state (existing `toolbarButton`); search-active indicator; sidebar row selection at 0.2 opacity (existing) |
| Destructive | `Color.red` via `role: .destructive` | Context-menu Delete actions (existing, unchanged) |
| Tertiary text | `.foregroundStyle(.tertiary)` | Count badge, empty hero subheadline, card subtitles |
| Secondary text | `.foregroundStyle(.secondary)` | Section header text, hero headline emphasis reduction, stat badges |

**Accent reserved for** (explicit list ‚Äî never "all interactive elements"):
1. Row-selection background (`Theme.sidebarSelectionBackground(isSelected:)` ‚Üí `Color.accentColor.opacity(0.2)`).
2. Active state on sidebar bottom-bar toolbar buttons (filter, search, show-active-sessions). Existing, unchanged.
3. Focus ring / keyboard focus (system default).

**Explicitly NOT accent-tinted in Phase 11:**
- The nested Bookmarks sub-header (matches the neutral treatment of every other `SidebarSectionHeader`).
- The overview empty hero icon (uses `.foregroundStyle(.secondary)` ‚Äî not `.tint` / `.accentColor`, because we don't want the empty state to scream).
- The overview empty hero text (no accent-tinted verbs/CTAs ‚Äî the subheadline mentions `+` as plain text, not as a colored chip).

**Color-related invariant:** the nested Bookmarks sub-section inherits the same list-row backgrounds and selection treatment as the outer Browsers section ‚Äî no custom fill, no indent-gutter color, no alternating shading. One visual rhythm across the Browsers block.

## Interaction Contracts (phase-specific)

Three new interaction patterns this phase introduces. Everything else is structural refactor of existing views.

### Pattern A ‚Äî Nested Bookmarks sub-section inside sidebar `BrowserSessionsSection`

**Rendering rule (per D-07, D-08, D-09, D-10, D-11):**

```
Browsers section renders when: !browsers.isEmpty || !visibleBookmarks.isEmpty

Content order inside section:
  1. ForEach(browsers) ‚Üí LocationBrowserRow    ‚Äî only if !browsers.isEmpty
  2. Nested Bookmarks sub-header                ‚Äî only if !visibleBookmarks.isEmpty
  3. If sub-header expanded:
       a. Unfiled bookmarks (ForEach BookmarkRow inline)
       b. Top-level folders (ForEach BookmarkFolderNode)
```

**Sub-header structure (inline, not via `SidebarSectionHeader`):**

| Slot | Spec |
|------|------|
| Container | `HStack(spacing: 4)` wrapped in `Button { ‚Ä¶ } .buttonStyle(.plain)` |
| Chevron | `Image(systemName: "chevron.right").font(.system(size: 9, weight: .semibold)).rotationEffect(.degrees(isExpanded ? 90 : 0))` |
| Label | `Text("Bookmarks")` at system default sidebar font |
| Count badge (collapsed only) | `Text("\(visibleBookmarks.count)").font(.system(size: 10)).foregroundStyle(.tertiary)` ‚Äî rendered only when `!isExpanded` |
| Trailing | `Spacer()` |
| Drop target | `.onDrop(of: [.text], delegate: BookmarkFolderDropDelegate(targetFolderID: nil, modelContext: modelContext))` ‚Äî matches existing root-level drop behavior on the top-level header |
| Toggle animation | `withAnimation(.snappy(duration: 0.15)) { isExpanded.toggle() }` ‚Äî same as `SidebarSectionHeader` |
| `@AppStorage` key | `"sidebar.browsers.bookmarks.expanded"`, default `true` (D-05) |

**Indent:** none (D-17-equivalent discretion for sidebar). The chevron glyph at the same leading edge as the outer `SidebarSectionHeader` chevron is sufficient visual cue; an added indent would push the nested label past the icon column of sibling rows and introduce a second gutter rhythm for no gain. Confirmed by checker dimension 5 (Spacing).

**Why not `SidebarSectionHeader`:** `SidebarSectionHeader` can't express count-only-when-collapsed without a parameter addition, and Phase 12 owns canonicalization of the shared header (UIX-20). Keeping the sub-header inline is a conscious temporary divergence; Phase 12 decides whether to promote this variant.

### Pattern B ‚Äî Nested Bookmarks sub-section inside overview Browsers card

**Rendering rule (per D-17, D-18, D-19, D-20):**

```
Browsers card renders when: !browserTabs.isEmpty || !projectBrowserBookmarks.isEmpty

Content order inside CollapsibleVStackSection("Browsers", ‚Ä¶):
  1. if !browserTabs.isEmpty {
       LazyVGrid of tab cards   ‚Äî existing grid, unchanged
     }
  2. if !projectBrowserBookmarks.isEmpty {
       Nested CollapsibleVStackSection("Bookmarks",
         storageKey: "overview.browsers.bookmarks.expanded.\(projectID)") {
           LazyVGrid of bookmark cards  ‚Äî migrated from old bookmarksContent
         }
     }
```

**Container:** reuse `CollapsibleVStackSection` ‚Äî not a custom chevron. `CollapsibleVStackSection` nests cleanly (it's a plain `VStack`); no re-entrance issues. Confirmed in 11-CONTEXT.md code_context.

**Indent inside the Browsers card:** none. The inner `CollapsibleVStackSection` sits at the same leading edge as the outer card content. Nesting is signaled by the visual containment (one card wrapper) and the second chevron ‚Äî not indentation. Per D-17 discretion note: "suggest none; the chevron alone signals hierarchy."

**Card migration details:**
- The old standalone `bookmarksSection` (`ProjectOverviewView.swift:346-353`) is deleted.
- The old `bookmarksContent` (`ProjectOverviewView.swift:355-428`) is split: the empty-branch VStack (`:358-367`) is deleted outright; the populated `LazyVGrid` branch (`:369-425`) migrates **verbatim** into the new nested `CollapsibleVStackSection`. `CardStyle`, column spec (`GridItem(.adaptive(minimum: 260, maximum: 400), spacing: 12)`), `prefix(12)` cap, and "Showing 12 of N" overflow caption all travel unchanged.

**Expansion keys (both preserved, independent):**
- Outer: `overview.browsers.expanded.\(projectID)` ‚Äî unchanged from today.
- Inner (new): `overview.browsers.bookmarks.expanded.\(projectID)` ‚Äî default `true`. Naming mirrors `overview.bookmarks.expanded.\(projectID)` but lives nested ‚Äî per D-21.

**Deprecation note:** the old `@AppStorage("overview.bookmarks.expanded.\(projectID)")` key becomes orphaned. Acceptable ‚Äî this is a localStorage-style preference key; leaving it stranded doesn't affect users (they won't see a stale key surface anywhere). Phase 12's `@AppStorage` audit can choose to delete-on-migration or leave alone.

### Pattern C ‚Äî Overview empty hero

**Trigger condition (per D-12, exact):**

```swift
terminalSessions.isEmpty
  && browserTabs.isEmpty
  && documentTabs.isEmpty
  && projectBrowserBookmarks.isEmpty
  && projectBookmarks.isEmpty
```

`incompleteTodos` / `completedTodos` are NOT factored in ‚Äî Todos always renders at top. The hero sits **below** the Todos section in the top-level body `VStack`.

**Structure:**

```
VStack(alignment: .center, spacing: 12) {
  Image(systemName: "square.grid.2x2")
    .font(.system(size: 48))
    .foregroundStyle(.secondary)
  Text("Nothing here yet")
    .font(.title3)
    .fontWeight(.semibold)
    .foregroundStyle(.secondary)
  Text("Press + to add your first Scratch Pad, Browser, Bookmark, or Location.")
    .font(.subheadline)
    .foregroundStyle(.tertiary)
    .multilineTextAlignment(.center)
}
.frame(maxWidth: .infinity)
.padding(.vertical, 48)
```

**Invariants:**
- No `+` button, no tappable affordance, no inline action. Hero is passive. User's action surface is the existing overview-header `+` (`UnifiedAddMenu` inside `ProjectOverviewView.newButton`). Enforces Phase 10's "exactly two `+` menus" rule.
- Horizontally centered; vertically positioned naturally below Todos (no forced centering in the scroll view ‚Äî the existing top-leading alignment is preserved, the hero just self-centers horizontally within its VStack slot).
- Multiline subheadline wraps at narrow widths; no fixed width.

### Delete-last-item behavior (Success Criteria #4, D-22)

- All section visibility is a function of `@Query` reactivity + `if !empty` guards. SwiftData emits updates automatically on `modelContext.save()` after delete.
- **No `withAnimation { }`**, no `.transition(‚Ä¶)`, no `.animation(‚Ä¶, value: count)` on the section/card show-hide branches.
- The chevron rotation inside `SidebarSectionHeader` (at `:19`) and `CollapsibleVStackSection` (at `:35`) keep their existing `withAnimation(.snappy(duration: ‚Ä¶))` ‚Äî those animate the expand/collapse of content that is still mounted, not the appearance/disappearance of the section itself.
- User-preference anchor: memory `feedback_no_slide_transitions.md` ("instant state swaps, no animated transitions for navigation").

## Accessibility & Keyboard

Non-regressive pass ‚Äî Phase 11 preserves every affordance already present. Specifics to double-check during execution / audit:

- **Nested Bookmarks sub-header button** is a `Button` (not `onTapGesture`) ‚Äî keyboard-focusable + VoiceOver-readable "Bookmarks, collapsed/expanded, button".
- **`BookmarkRow`** keeps its existing `.modifier(DeleteKeyHandler(enabled: isHovering, action: delete))` ‚Äî Delete-key on hover still works in the nested sub-section.
- **`BookmarkFolderNode`** keeps `DisclosureGroup` + Button label pattern per `coding-patterns.md ¬ß4` (no `onTapGesture` conflict).
- **Nested Bookmarks sub-header chevron** lives inside the same Button as the label ‚Äî one tap target, not two. VoiceOver reads the combined element once.
- **Overview empty hero** has no interactive elements ‚Äî nothing to focus, nothing to tab to. VoiceOver reads it as static text ("Nothing here yet. Press plus to add your first Scratch Pad, Browser, Bookmark, or Location.").
- **No keyboard shortcut changes.** File menu shortcuts (`Cmd+Option+B` etc.) and the focus-scoped browser shortcuts stay unchanged per 10-CONTEXT.md D-13.

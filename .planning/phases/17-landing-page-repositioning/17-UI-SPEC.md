### Story section background tint (Claude's Discretion â€” LOCKED)

Use `bg-card` (`#18181B`) with an explicit `border-y border-border-subtle` top and bottom edge.

Rationale: `#18181B` is already the site's "elevated surface" token â€” it reads visually distinct from `#0F0F12` without introducing a new color. Using `bg-card` directly (not a Tailwind opacity modifier) keeps the implementation to a single class that the existing palette already defines. The border-y provides rhythm without requiring a new visual language. Do NOT use `bg-zinc-*`, `bg-neutral-*`, or any opacity modifier â€” they are not in the defined token set.

Full story section container classes: `bg-card border-y border-border-subtle`

### Secondary CTA visual style (Claude's Discretion â€” LOCKED)

Use the **outline button** style. Not a text-link.

Rationale: The existing page already has two CTA tiers:
1. Primary: `bg-linear-to-b from-teal to-teal-dark` (solid teal gradient)
2. Secondary: bare `text-secondary text-sm hover:text-primary` (text-only ghost link)

The "text-only" style currently reads as a footnote ("try free â€” 3 day trial") rather than a co-equal CTA. For the OSS flip, `Star on GitHub` needs to read as a genuine secondary action â€” not a footnote. An outline button makes the hierarchy clear: "Here are two things you can do" vs "primary action / footnote."

**Outline button classes (hero size):**
```
inline-flex items-center justify-center gap-2 rounded-lg border border-border px-8 py-4 text-base font-semibold text-primary hover:border-teal-dark hover:text-teal transition-all
```

**Outline button classes (header/compact size):**
```
inline-flex items-center justify-center gap-2 rounded-lg border border-border px-4 py-2 text-sm font-medium text-primary hover:border-teal-dark hover:text-teal transition-all
```

The hover state (`hover:border-teal-dark hover:text-teal`) uses existing palette tokens and makes the button feel interactive without stealing the primary accent.
